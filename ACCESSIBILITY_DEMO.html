🔍 What is @axe-core/playwright?

axe-core is an open-source accessibility testing library developed by Deque Systems — a leading company in web accessibility. It integrates directly with Playwright to automatically scan a page's DOM and detect violations against the WCAG standard.


Purpose: Ensure web applications are usable by everyone — including blind users relying on screen readers, people who navigate by keyboard only, and users with colour-vision deficiencies.

1

Playwright opens the page

Navigate to the URL and perform any required steps (login, click, etc.)

2

axe-core scans the DOM

Runs ~100 rules against every HTML element on the page

3

Violations are classified

Each issue is rated: critical, serious, moderate, or minor

4

Test passes or fails

Test fails if violations exceed the configured threshold

[critical] select-name: Ensure select element has an accessible name
const results = await new AxeBuilder({ page }) .withTags(['wcag2a', 'wcag2aa', 'wcag21a', 'wcag21aa']) .analyze(); const critical = results.violations.filter(v => v.impact === 'critical'); expect(critical).toHaveLength(0); // ❌ FAIL: critical = 1 violation (select-name)
💡 Correct fix: Add aria-label="Sort products by" to the <select> element.
This is a bug in saucedemo.com itself — axe-core catches an issue that is invisible during a normal code review.
